# Contributing to FormCN

Thank you for your interest in contributing to FormCN! We welcome contributions of all kinds, from bug fixes and documentation updates to new features and templates.

## Getting Started

### Prerequisites

- Node.js 18+
- pnpm (recommended) or npm/yarn

### Setup

1.  **Fork the repository** on GitHub.
2.  **Clone your fork**:
    ```bash
    git clone https://github.com/YOUR_USERNAME/formcn.git
    cd formcn
    ```
3.  **Install dependencies**:
    ```bash
    pnpm install
    ```
4.  **Start the development server**:
    ```bash
    pnpm dev
    ```
    This starts the main web application (builder + docs) at `http://localhost:3000`.

## Development Workflow

### Adding a New Template

1.  **Define the Template**:
    Create a new TypeScript file in `apps/web/src/registry/default/templates/` (e.g., `my-new-form.ts`).
    Implement the `FormTemplate` interface.

2.  **Export**:
    Add your template to `apps/web/src/registry/default/templates/index.ts`.

3.  **Create Static Component**:
    Create a corresponding `.tsx` file (e.g., `my-new-form-form.tsx`) in the same directory. This is the code that users will copy/install.
    Ensure it matches the logic generated by the builder.

4.  **Register**:
    Add an entry to `apps/web/registry.json`.

5.  **Build Registry**:
    Run the registry build script to update the public API files (`apps/web/public/r/*.json`).
    ```bash
    pnpm --filter web registry:build
    ```

### Modifying the Generator

The core logic for generating form code resides in `apps/web/src/registry/default/lib/form-generator.ts`.
Any changes here affect both the "Code" tab in the builder and the code generated by `shadcn add`.

After modifying the generator:
1.  Verify changes in the Visual Builder (`http://localhost:3000/editor`).
2.  Run `pnpm --filter web registry:build` to update artifacts.

## Pull Request Guidelines

1.  Create a descriptive branch name (e.g., `feat/multi-step-wizard`, `fix/input-validation`).
2.  Commit your changes with clear messages.
3.  Ensure the registry build passes.
4.  Open a Pull Request against the `main` branch.
