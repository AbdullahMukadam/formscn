# Contributing to FormsCN

Thank you for your interest in contributing to FormsCN! We welcome contributions of all kinds, from bug fixes and documentation updates to new features and templates.

## Getting Started

### Prerequisites

- Node.js 18+
- pnpm (recommended) or npm/yarn

### Setup

1.  **Fork the repository** on GitHub.
2.  **Clone your fork**:
    ```bash
    git clone https://github.com/AbdullahMukadam/formscn.git
    cd formcn
    ```
3.  **Install dependencies**:
    ```bash
    pnpm install
    ```
4.  **Start the development server**:
    ```bash
    pnpm dev
    ```
    This starts the main web application (builder + docs) at `http://localhost:3001`.

## Roadmap & Areas for Contribution

We are actively looking for help with:

*   **Conditional Logic**: Adding `if/then` branching to the visual builder and schema generator.
*   **Drag-and-Drop Step Reordering**: Improving the UX for multi-step form management.
*   **AI Integration**: Generating form schemas from text prompts.
*   **New Templates**: Adding more complex real-world forms (e.g., KYC, Medical Intake).

## Development Workflow

### Adding a New Template

The easiest way to contribute a new template is to build it visually first.

1.  **Build Visually**:
    Open the [Visual Builder](http://localhost:3001/editor) locally. Build your form (fields, steps, validation).

2.  **Create Template Definition**:
    Create a new file in `apps/web/src/registry/default/templates/` (e.g., `medical-intake.ts`).
    
    ```ts
    import type { FormTemplate } from "../types";

    export const medicalIntake: FormTemplate = {
      id: "medical-intake",
      name: "Medical Intake",
      description: "Patient registration and medical history.",
      fields: [
         // ... your fields here
      ],
    };
    ```

3.  **Register the Template**:
    Add it to `apps/web/src/registry/default/templates/index.ts`.

4.  **Build Registry**:
    Run the registry build script to generate the artifacts.
    ```bash
    pnpm --filter web registry:build
    ```

### Modifying the Generator

The core logic for generating form code resides in `apps/web/src/registry/default/lib/form-generator.ts`.
Any changes here affect both the "Code" tab in the builder and the code generated by `shadcn add`.

After modifying the generator:
1.  Verify changes in the Visual Builder (`http://localhost:3001/editor`).
2.  Run `pnpm --filter web registry:build` to update artifacts.

## Pull Request Guidelines

1.  Create a descriptive branch name (e.g., `feat/multi-step-wizard`, `fix/input-validation`).
2.  Commit your changes with clear messages.
3.  Ensure the registry build passes.
4.  Open a Pull Request against the `main` branch.
